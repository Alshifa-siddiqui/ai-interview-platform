# AI Interview Platform

An AI-powered mock interview platform where candidates answer role-specific questions generated by Claude, receive instant scoring, and track their progress over time.

## Features

- **AI-generated questions** — Claude creates tailored interview questions based on role and difficulty level
- **Resume-aware questions** — Upload a PDF or DOCX resume and Claude tailors every question to your actual experience
- **Structured evaluation** — Each answer is scored out of 10 with strengths, areas to improve, and detailed feedback
- **Progress tracking** — Score history chart shows your improvement over time across multiple interviews
- **Dark mode** — Toggle between light and dark themes; preference persists across sessions
- **Supabase auth** — Email/password sign-up and sign-in with JWT-secured API routes

## Local Setup

### Prerequisites

- Python 3.11+
- Node.js 18+
- A [Supabase](https://supabase.com) project
- An [Anthropic API key](https://console.anthropic.com)

### 1. Clone the repo

```bash
git clone https://github.com/Alshifa-siddiqui/ai-interview-platform.git
cd ai-interview-platform
```

### 2. Backend

```bash
cd backend
pip install -r requirements.txt
```

Create `backend/.env`:

```env
ANTHROPIC_API_KEY=sk-ant-...
DATABASE_URL=postgresql://postgres:<password>@db.<project>.supabase.co:5432/postgres
SECRET_KEY=<random 32-byte hex string>
SUPABASE_JWT_SECRET=<your Supabase JWT secret>
```

Start the backend:

```bash
uvicorn app.main:app --reload --port 8080
```

### 3. Frontend

```bash
cd frontend
npm install
```

Create `frontend/.env`:

```env
VITE_SUPABASE_URL=https://<project>.supabase.co
VITE_SUPABASE_ANON_KEY=<your Supabase anon key>
```

Start the frontend:

```bash
npm run dev
```

The app runs at **http://localhost:5174** and proxies `/api` calls to the backend at port 8080.

### 4. Run both at once (Windows)

From the project root:

```bat
run-stack.bat
```

## Environment Variables

### `backend/.env`

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | Yes | Anthropic API key — get one at console.anthropic.com |
| `DATABASE_URL` | Yes | PostgreSQL connection string (Supabase) |
| `SECRET_KEY` | Yes | Random secret used for internal signing — run `python -c "import secrets; print(secrets.token_hex(32))"` |
| `SUPABASE_JWT_SECRET` | Yes | Found in Supabase → Project Settings → API → JWT Secret |

### `frontend/.env`

| Variable | Required | Description |
|---|---|---|
| `VITE_SUPABASE_URL` | Yes | Your Supabase project URL |
| `VITE_SUPABASE_ANON_KEY` | Yes | Your Supabase anon/public key |

## Deployment

- **Backend** — Deploy the `backend/` folder to [Render](https://render.com) or [Railway](https://railway.app). The `Procfile` is already configured. Set the environment variables from the table above.
- **Frontend** — Deploy the `frontend/` folder to [Vercel](https://vercel.com) or [Netlify](https://netlify.com). Set `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`, and update `vite.config.ts` proxy target to your deployed backend URL.
